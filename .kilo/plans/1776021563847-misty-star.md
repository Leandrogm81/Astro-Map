# Plan: Corrigir orientação invertida de slice da roda do zodíaco

## Contexto
O usuário observou que um elemento `<path>` da roda do zodíaco está invertido:
```html
<path d="M 179.5703646533711 90.46684206542233 A 380 380 0 0 0 363.86891501016805 21.721604241720513 L 368.6230051404091 71.49507736780993 A 330 330 0 0 1 208.5742640410854 131.1948891620773 Z" fill="#3b82f615" stroke="#3b82f6" stroke-width="1.5"></path>
```
Esse caminho corresponde a uma fatia da roda do zodíaco (sinal do ar) que está sendo desenhada na direção errada, resultando em um aspecto “invertido”.

## Objetivo
Ajustar a geração do caminho SVG (`<path>`) no componente **TraditionalChart** para que as fatias da roda do zodíaco sejam desenhadas na orientação correta, mantendo as cores e estilos atuais.

## Estratégia
1. **Analisar a função `getArcPath`** que constrói o caminho SVG.
2. **Identificar o motivo da inversão** – provavelmente o flag de varredura (`sweep-flag`) do comando `A` está incorreto ou a ordem dos pontos está trocada.
3. **Propor correção**:
   - Trocar o `sweep-flag` do arco externo de `0` para `1` (ou inverter a ordem de `startAngle`/`endAngle`).
   - Garantir que o arco interno use o flag correto (`0` ou `1` conforme necessário) para fechar a forma sem cruzamento.
4. **Implementar a alteração** na função `getArcPath`.
5. **Testar visualmente**:
   - Executar `npm run dev`.
   - Verificar a roda do zodíaco nas páginas *Mapa* e *Tradicional*.
   - Confirmar que a fatia em questão (e todas as demais) está orientada corretamente.
6. **Revisar código** para garantir que a mudança não afete outros componentes (por exemplo, o `AstroChart` que usa lógica similar).
7. **Executar testes automatizados** (se houver) para garantir que nada quebre.
8. **Commit** da correção com mensagem explicativa.
