# DOCS FLOW

**Descrição:**  
Use este workflow quando o objetivo for criar documentação em Markdown sobre um sistema, app, projeto, módulo, fluxo, integração, regra ou processo.  
A função deste workflow é transformar informações fornecidas pelo usuário em um documento `.md` claro, estruturado e pronto para salvar.

---

## 🎯 Missão

Transformar informações sobre um sistema ou projeto em documentação Markdown útil, organizada e reaproveitável.

Este workflow existe para:

- criar documentação técnica ou interna
- organizar informações soltas em documento
- gerar guias, manuais, referências e resumos
- estruturar conteúdo para consulta futura
- produzir arquivos `.md` prontos para copiar e salvar

---

## 🧭 Princípios

- **Gerar somente documentação.**
- **Não executar código.**
- **Não alterar arquivos do projeto.**
- **Não implementar funcionalidades.**
- **Não inventar detalhes que o usuário não forneceu.**
- **Quando faltar informação, marcar claramente como pendente.**
- **A saída deve ser útil, limpa e pronta para salvar como `.md`.**

---

## 📥 Quando Acionar

Acione este workflow quando quiser:

- criar um documento `.md`
- documentar um sistema
- documentar um app
- montar um guia técnico
- criar um README
- criar documentação interna
- organizar workflows em documento
- documentar módulos, fluxos, integrações ou regras
- transformar tópicos soltos em Markdown
- montar um manual, checklist, tutorial ou referência

---

## 📥 Entradas Esperadas

Sempre que possível, informar:

- tema do documento
- assunto ou área do sistema
- itens obrigatórios
- nível de detalhe desejado
- público-alvo do documento
- nome sugerido do arquivo, se houver

Exemplos de tema:

- arquitetura do app
- lista de workflows
- módulos principais
- integrações
- regras de negócio
- convenções do sistema
- fluxo de validação
- documentação de componentes

## 🛠️ Protocolo de Execução

### Passo 1: Interpretar o Pedido

Identificar:

- tipo de documento
- tema principal
- público-alvo
- seções obrigatórias
- tom desejado
- nível de profundidade
- nome do arquivo, se existir

Se o pedido estiver incompleto, usar apenas a clarificação mínima necessária.

---

### Passo 2: Delimitar o Escopo do Documento

Definir:

- o que o documento deve cobrir
- o que o documento não precisa cobrir
- se o foco é visão geral, referência, guia, manual ou checklist
- se o conteúdo será mais técnico, mais operacional ou mais introdutório

---

### Passo 3: Montar a Estrutura

Organizar o documento em uma hierarquia lógica.

Exemplo base:

```markdown
# Título

## Visão Geral

## Objetivo

## Escopo

## Conteúdo Principal

## Regras / Estrutura / Fluxos

## Observações

## Próximos Passos
```
