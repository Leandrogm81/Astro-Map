# PROJECT CANVAS - AstroMap

**Descrição:**  
Use este workflow quando a ideia ainda estiver em estágio inicial, ampla ou parcialmente formada.  
O objetivo é organizar visão, problema, público, proposta de valor, escopo inicial e riscos em um formato rápido e claro, antes de acionar o `SPEC STUDIO`.

---

## 🎯 Missão

Transformar uma ideia difusa em um mapa simples e utilizável de projeto.

Este workflow existe para:

- organizar pensamento inicial
- reduzir nebulosidade
- identificar problema, usuário e proposta de valor
- delimitar uma direção inicial
- decidir se a ideia já está pronta para `DISCOVERY MODE`, `PRD MODE`, `MVP MODE` ou `SPEC MODE`

---

## 🧭 Princípios

- **Mapa antes de documento longo.**
- **Clareza inicial vale mais que detalhe precoce.**
- **Não discutir implementação cedo demais.**
- **Canvas serve para orientar, não para executar.**
- **Projeto confuso precisa de estrutura leve antes de especificação pesada.**

---

## 📥 Quando Acionar

Acione este workflow quando você quiser:

- organizar uma ideia nova
- pensar um projeto do zero
- resumir um conceito antes do PRD
- estruturar uma visão inicial
- validar se a ideia já tem direção suficiente
- sair de brainstorming e entrar em planejamento real

---

## 📥 Entradas Esperadas

Sempre que possível, informar:

- ideia inicial
- problema percebido
- tipo de projeto
- para quem isso serviria
- contexto geral
- motivação
- referências, prints, links ou exemplos
- limitações conhecidas, se houver

Mesmo com pouco material, o workflow deve conseguir estruturar a visão inicial.

---

## 📤 Saídas Possíveis

Este workflow pode gerar:

- **Project Canvas**
- **Idea Snapshot**
- **Initial Scope Map**
- **Discovery Brief**
- **Open Questions List**
- **Recomendação de próxima rota no SPEC STUDIO**

---

## 🛠️ Protocolo de Execução

### Passo 1: Resumir a Ideia

Transformar a ideia em uma frase simples:

> “Queremos criar / melhorar / testar [X] para [Y].”

Se a frase ainda estiver confusa, simplificar antes de continuar.

---

### Passo 2: Declarar o Problema

Responder:

- qual problema ou necessidade motivou essa ideia?
- esse problema é real, recorrente ou só interessante?
- o que acontece hoje sem essa solução?

---

### Passo 3: Identificar o Usuário

Mapear:

- usuário principal
- usuário secundário, se houver
- contexto de uso
- por que isso importaria para esse usuário

---

### Passo 4: Definir a Proposta de Valor

Responder:

- que valor essa ideia entrega?
- o que melhoraria para o usuário?
- qual benefício mais claro ela promete?

---

### Passo 5: Delimitar Escopo Inicial

Registrar:

- o que parece fazer parte do projeto
- o que provavelmente está fora
- o que ainda é cedo demais para decidir

Não tentar fechar detalhes técnicos neste momento.

---

### Passo 6: Identificar Restrições e Dependências

Checar, quando aplicável:

- tempo
- custo
- integração
- dados
- conhecimento de domínio
- dependências externas
- limitações técnicas já conhecidas

---

### Passo 7: Mapear Riscos e Dúvidas

Registrar:

- principais incertezas
- riscos de entendimento
- pontos nebulosos
- decisões que ainda não podem ser tomadas

---

### Passo 8: Escolher a Próxima Rota

Ao final, decidir qual é o próximo passo mais útil:

- `SPEC STUDIO em DISCOVERY MODE`
- `SPEC STUDIO em PRD MODE`
- `SPEC STUDIO em MVP MODE`
- `SPEC STUDIO em SPEC MODE`
- `Open Questions List`, se ainda faltar clareza

---

## 🧱 Estrutura Recomendada de Saída

Quando a saída for **Project Canvas**, usar preferencialmente esta estrutura:

1. Título da ideia
2. Resumo da ideia
3. Problema principal
4. Usuário principal
5. Proposta de valor
6. Escopo inicial
7. Fora de escopo inicial
8. Restrições e dependências
9. Riscos e dúvidas
10. Próximo passo recomendado

---

## 🚫 Fronteiras

Este workflow:

- **não implementa**
- **não escreve código**
- **não gera arquitetura detalhada**
- **não substitui PRD ou Spec**
- **não inicia BUILD FLOW**
- **não atualiza MAESTRO**
- **não cria log por reflexo**

---
