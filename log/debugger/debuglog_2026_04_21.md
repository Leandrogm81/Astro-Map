# Debug Log - 2026-04-21

## Problema Identificado

Erro `SyntaxError: Unexpected token 'I', "Internal S"... is not valid JSON` ao gerar relatórios de IA.

### Causa Raiz

1. **IDs de Modelos Incorretos**: A API estava tentando usar IDs de modelos que não existem mais ou estavam mal formatados na OpenRouter (ex: `gemini-2.0-flash-lite`).
2. **Tratamento de Erro Inadequado**: Quando a OpenRouter falhava (401/404), o Next.js retornava uma página de erro HTML ou a string "Internal Server Error", que o frontend tentava parsear como JSON via `response.json()`. Como a string começa com "I", o parser lançava o erro "Unexpected token I".
3. **Lógica de Prompt Solar**: O campo `solarRevolution` no body da requisição não estava sendo mapeado corretamente para `solarChart` no backend.

## Correções Aplicadas

### Backend (`src/app/api/report/route.ts`)

- **Atualização de Modelos**: IDs atualizados para versões estáveis (`google/gemini-flash-1.5`, `deepseek/deepseek-chat`, `qwen/qwen-2.5-72b-instruct`).
- **Resiliência de Parsing**: Adicionado `try-catch` no parsing do body para evitar crashes imediatos.
- **Respostas JSON Estritas**: Todos os erros agora retornam explicitamente um objeto JSON `{ error: '...' }` com o status code correto.
- **Refinamento de Stream**: O processador de stream agora trata chunks incompletos de forma silenciosa e garante o encoding UTF-8.
- **Modos de Relatório**: Corrigida a lógica de seleção de prompts para `natal`, `solar` (Revolução) e `traditional`.

### Frontend (`src/components/AIReport.tsx`)

- **Validação de Content-Type**: Antes de chamar `response.json()`, o componente verifica se o cabeçalho é `application/json`. Se for texto, lê como texto para exibir a mensagem de erro real em vez do erro de sintaxe.
- **Correção de Payload**: Campo `solarChart` enviado corretamente conforme esperado pelo backend.

A API agora é resiliente a falhas externas e fornece mensagens de erro claras ao usuário.
